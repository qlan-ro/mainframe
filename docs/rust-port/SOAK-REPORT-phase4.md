## Post-review fix analysis (adversarial findings)

Three adversarial-review findings targeted this soak; all three moved the deltas from
`-2` to `-1` and eliminated the structural chat.updated divergence:

- **Adapter connect-replay (was: 0 replay events).** `handle_socket` now streams
  `build_connect_replay_events(ctx.adapter_registry.get_snapshots())` right after
  `connection.ready`, mirroring `buildConnectReplayEvents`. Rust now replays every
  **probed** adapter — visible as the leading `adapter.models.updated` on every
  scenario (previously absent).
- **`adapter.models.updated` broadcast (claimed missing).** Refuted: `apply_refresh`
  already emits it (`lib.rs`). The soak's `-2` was entirely the missing connect-replay
  above; the live probe broadcast fires at boot, before any client connects, so only
  the replay is observable to a scenario client.
- **`chat.updated` finalization ordering.** `send_message` was **awaiting**
  `do_generate_title` (a CLI shell-out); TS fires it `.catch()`-style without awaiting.
  Now spawned. The title `chat.updated` moved out of the early cluster: parity-text and
  interrupt now match Node exactly (`… chat.contextUsage chat.updated` trailing, two
  early updates). tool-permission lands the async title update mid-turn instead of
  trailing — inherent nondeterminism of a fire-and-forget shell-out (the chat.updated
  **count** matches Node 4↔4); this now mirrors Node's own async title semantics.

**Residual `-1` (all scenarios): codex catalog stays `fallback` in Rust.** Node replays
two `adapter.models.updated` (claude + codex); Rust replays one (claude). Standalone the
Rust daemon leaves codex at `catalogSource: fallback / 0 models` even after 45s despite
`codex` being installed — `CodexAdapter::list_models` (temp app-server + `model/list`)
yields no live catalog in this environment, so codex is never `probed` and is correctly
excluded from the replay. This is a separate codex-adapter probe matter, pre-existing and
outside the connect-replay/ordering findings; the replay wiring itself is faithful (it
replays exactly the probed adapters, as Node does).
