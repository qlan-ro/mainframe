## ‚≠ê Golden rule: chat/thread/thread-list components ‚Äî assistant-ui first, **stop-and-ask on mismatch**

For **any chat / thread / thread-list / message / tool-card / permission / composer component**, follow this process ‚Äî never skip it:

1. **Research** what assistant-ui provides for it (check the *installed source*, not just the docs ‚Äî they drift).
2. **Compare** it to our warm-chrome prototype design (the relevant artboard).
3. **If it matches** ‚Üí use the assistant-ui component, **restyled** to the design. Do not rebuild what it provides.
4. **If it does NOT match** ‚Üí **STOP. Build neither version yet.** Present a concise **side-by-side summary ‚Äî our design vs the native assistant-ui component** (what each offers, where they diverge, the cost of each direction) and **ask for a decision.**

Never silently (a) rebuild what assistant-ui already provides, nor (b) force a native component that doesn't fit the design ‚Äî a mismatch is a **human decision**. *(Subagents can't prompt the user: on a mismatch, STOP, build nothing, and return the design-vs-native summary to the orchestrator, who asks.)*

## Runtime & data
- **`useExternalStoreRuntime`** fed by a **per-chat controller** (the `react-opencode` shape: controller + pure reducer + `handle-daemon-event` + projection). **Not** AssistantTransport.
- **No app-side message cache.** The daemon is the single source of truth; the controller holds only the per-chat projection, disposed on switch. Drift is handled by **refetch-on-gap** (unknown-id delta ‚Üí REST re-seed), not a client cache or a daemon `seq`.
- **Optimistic-send reconcile = count-aware + server-authoritative** (judo-A). One matcher clears optimistic pendings against confirmed server user-messages via a multiset: each server copy clears at most one pending, oldest-first. The live `message.added` passes a single message; the history re-seed passes the full list. **No time window, no empty-text wildcard, no over-clearing** of legitimate duplicate sends.
- Sessions list via **`useRemoteThreadListRuntime`** + `chats-remote-adapter` (step 11, built). Global runtime mounted ONCE in App.tsx (App is the real root: DaemonPortProvider ‚Üí AssistantRuntimeProvider). `useChatRuntimeHook` resolves per-thread context via `useAuiState((s) => s.threadListItem)` and keys the controller by the stable `item.id`; WS sub gated to the active thread (isActive = mainThreadId === item.id && item.remoteId != null). WS list side-effects (reload/unread/cross-project-filter-clear/archived-active-fallback) run via `useSessionListRouter()` under the provider.

## Architecture
- **Feature-first** under `features/`; `layout/` + `surfaces/` only *compose* features into the typed surfaces (Chat/Files/Run). No feature imports `layout/`.
- **Surface intent, not reach-through:** a feature that needs to open a file/diff/surface emits an intent; never `someStore.getState().openX()`.
- **`lib/tauri/` is the only Tauri-aware module** ‚Äî all `window.mainframe.*` equivalents live there (Rust commands/events).
- **Pure logic** (convertMessage, diff math, file-types) goes in a shared bundleable location, **not** the `mainframe-core` *sidecar process*.

## Component layer & theme
- **shadcn/ui**, not raw Radix. Build the `components/ui/` primitives once; features compose them.
- Theme via `mainframe-theme.css` tokens. **Token traps:** never use the `/opacity` modifier on CSS-var colors; use only real `mf-*` token names.

## Conventions
- **No file > 300 lines**, no function > 50. Decompose god-files on port (don't carry them).
- **Every interactive element needs a stable `data-testid`** (`<surface>-<element>`); `ui/` primitives stay passthrough.
- **`convertMessage` invariants are load-bearing:** WS14c dual re-encode (top-level AND nested in task_group), the `\0` permission sentinel, per-message `uniqueId()` dedup, ‚â•1-content-part + error fallbacks.
- **Pin the whole `@assistant-ui/*` set together at the `react@0.14.14` / `core@0.2.10` line** (matches `react-opencode`'s `^0.14.14` peer; has `groupPartByType` + `display:'standalone'` + the `GroupedParts` `indicator` slot; fixes the `core@0.2.2`‚Üî`store@0.2.10` skew). They're independently versioned and sub-1.0 ‚Äî never let `^` ranges drift them apart.
- **Drop, don't port,** obsolete code: the `zone/` layout, layout/ui stores, the dual tool dispatcher, regex go-to-definition.

## Daemon contract
The WS/REST contract is **co-owned by the mobile submodule** ‚Äî changes must be **additive** (mobile ignores unknown fields). No unilateral reshaping.
